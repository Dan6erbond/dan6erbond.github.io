MelonðŸ”—

Melon is a general utility bot ranging from utilities such as reaction-roles and message management to Melons (and upcoming Tags) to make it a portable knowledge database.

Melon's initial feature-set was limited to reaction-roles and management as the bot served as a utility bot for the use of Mariavi and was capable of advertising certain projects. The addition of Melons makes the bot an ideal companion for servers based around productivity as the Melons are split in to categories which can be enabled and disabled by server administrators.

Reaction-RolesðŸ”—

Reaction-roles can be setup by administrators of a server and allow the users to click on reactions of a certain message to receive a role. The command !addreactrole [emoji, role, message, channel] will have Melon add a reaction with the [emoji] of your choice to the messsage, as well as an entry to her database that creates a link between the message's reaction and a given role.

Removing a Reaction-RoleðŸ”—

Reaction-roles can be removed by using the command !remreactrole [emoji, message, channel] after calling it Melon will remove the reaction from the message as well as the link to avoid assigning the reaction in the future.

Reactions beneath the message can be clicked on for the user to receive a role.

MessagesðŸ”—

Melon has two handy commands that allow you to quote people, evaluate poll results and more!

!quote [id, breakdown]: This command will return a message in the form of an embed. The id of the message must be fed, whereas the boolean parameter breakdown is optional. If a message has reactions added to it, they will be shown as part of the embed Melon returns quoting the message. Otherwise an embed with the author's name, a URL to the message as well as the text (including markdown) will be returned:

Melon will return the message info including reactions.

!rawmsg [id]: !rawmsg follows a similar design pattern to !quote and will return the message within a simple code block allowing you to see the markdown which proves useful for when a message needs to be posted somewhere else and you aren't the author.

ManagementðŸ”—

Given you have the Manage Messages permission in the channel you are trying to use either of the following commands, Melon allows you bulk remove messages:

!nuke: Removes a maximum of 1000 messages in the current channel, ignores pinned messages.
!clear [amt]: Removes the given amount of messages, again ignoring pinned messages.

MelonsðŸ”—

Melons are a way of storing information that derrived from the Tags Substarter's bot, Substar, used to manage. Currently Melons are solely managed by Amy and Ravi and split in to a couple categories, amongst them programming and reddit.

Searching through the Melons is easy, type !melon [search] and Melon will return any findings or display exact matches:

Clicking on any of the numbers will display the appropriate Melon.

The information is then displayed in an orderly fashion.

Server administrators can view the currently existing Melon categories by using the command !meloncats and en-/disable a particular one with !enablecat [cat] and !disablecat [cat] respectively. This gives a lot of control over what kind of information is shared and prioritized. The search engine powering Melon is Fuzzle and allows for users to search through tags which are added by Mariavi as well to supercharge the experience.

UtilityðŸ”—

Alongside the previously elaborated commands Melon has some miscellaneous utility commands which have been proven useful in the past:

!color [amt, tolerance]: Melon will find the most previous image attachment and return a color grid with the RGB and HEX values of the most common colors.
In addition, the amt value can be fed to specify how many colors the grid should contain and tolerance should be a value between 3-45 to specify how close the colors should be to one another. Higher tolerance values means the colors will be more unique than if they are closer to the default value of 6.

!avatar [user]: This command will return the given user's (if not specified, your own) avatar in either a png or URL format (for animated avatars).

!rand [options]: If the option parameter follows the format [int]-[int] Melon will generate a random number, otherwise she will split the options with a space between them and return a random pick.
