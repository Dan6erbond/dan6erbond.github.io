Melonüîó

Melon is a general utility bot ranging from utilities such as reaction-roles and message management to Melons (and upcoming Tags) to make it a portable knowledge database.

Melon's initial feature-set was limited to reaction-roles and management as the bot served as a utility bot for the use of Mariavi and was capable of advertising certain projects. The addition of Melons makes the bot an ideal companion for servers based around productivity as the Melons are split in to categories which can be enabled and disabled by server administrators.

Reaction-Rolesüîó

Reaction-roles can be setup by administrators of a server and allow the users to click on reactions of a certain message to receive a role. The command !addreactrole [emoji, role, message, channel] will have Melon add a reaction with the [emoji] of your choice to the messsage, as well as an entry to her database that creates a link between the message's reaction and a given role.

Reactions beneath the message can be clicked on for the user to receive a role.

Removing a Reaction-Roleüîó

Reaction-roles can be removed by using the command !remreactrole [emoji, message, channel] after calling it Melon will remove the reaction from the message as well as the link to avoid assigning the reaction in the future.

Messagesüîó

Melon has two handy commands that allow you to quote people, evaluate poll results and more!

!quote [id, channel]: This command will return a message in the form of an embed. The id of the message must be fed, whereas the boolean parameter breakdown is optional. If a message has reactions added to it, they will be shown as part of the embed Melon returns quoting the message. Otherwise an embed with the author's name, a URL to the message as well as the text (including markdown) will be returned:

Melon will return the message info including reactions.

!rawmsg [id, channel]: !rawmsg will return the message within a simple code block allowing you to see the markdown which proves useful for when a message needs to be posted somewhere else and you aren't the author.

Managementüîó

Given you have the Manage Messages permission in the channel you are trying to use either of the following commands, Melon allows you bulk remove messages:

!nuke: Removes a maximum of 1000 messages in the current channel, ignores pinned messages.
!clear [amt]: Removes the given amount of messages, again ignoring pinned messages.

Melonsüîó

Melons are a way of storing information that derrived from the Tags Substarter's bot, Substar, used to manage. Currently Melons are solely managed by Amy and Ravi and split in to a couple categories, amongst them programming and reddit.

Searching through the Melons is easy, type !melon [search] and Melon will return any findings or display exact matches:

Clicking on any of the numbers will display the appropriate Melon.

The information is then displayed in an orderly fashion.

Server administrators can view the currently existing Melon categories by using the command !meloncats and en-/disable a particular one with !enablecat [cat] and !disablecat [cat] respectively. This gives a lot of control over what kind of information is shared and prioritized. The search engine powering Melon is Fuzzle and allows for users to search through tags which are added by Mariavi as well to supercharge the experience.

Tagsüîó

Tags are a subset of Melons in the sense that they can be configured on a per-guild basis but still run through the Melon framework to deliver the information. To enable the Tag feature a server admin will need to run the command !enabletags upon which Melon will create a role called üçâ Tag Role whose name can be changed, and it can be given to anyone that should have write access to the server's Tags.

If the role is deleted and you would like to re-enable the feature, just call !enabletags again.

Anyone with the role can now use !addtag [key], !deltag [key] as well as !edittag [key] to perform actions on the particular server's database.

Viewing a Tag is done over the Melon framework which means that you can call !melon [search] as usual and Melon will scan through both the enabled Melon categories as well as the server's Tags to display all the results that were found.

Automojiüîó

Automojis are simple reactions that Melon will add to every message written into a certain channel!

Using !addemoji [emoji] in the channel you want to have the setup, Melon will first add the emoji to every message made in the past and then create an entry in her database so that every message made in the future gets the emoji. This is especially useful for to-do lists and channels with a voting system. If you try to add an emoji that isn't valid, she will let you know and if the emoji is already setup, the same thing.

To remove an emoji you can simply use the command !remoji [emoji] that will retroactively remove all the emojis previously added as well as the entry in the database, if nothing was found, you'll be notified.

Automojis allow channels to have persistent emojis and aren't limited to messages by the bot.

Pollüîó

Melon has a couple of commands and features that allow users to setup poll channels as well as evaluate the results of a poll after it has taken place.

One command is !poll [id, channel] which works similarly to !quote, in terms of what arguments it takes, but instead of a minified version of the reactions you see a full breakdown of who pressed what reaction:

The !poll command shows you a breakdown of all the reactions pressed in a poll.

In addition to using !poll to see a breakdown of the reactions, channels can be setup as poll channels. Melon will then check if a message sent/edited in the channel has any emojis (certain emojis aren't supported yet) and add those as reactions, if none were found, she will add some default emojis that can be configured, but are set to üëç and üëé in the beginning.

With the command !togglepoll the feature can be enabled/disabled. If you want custom generic poll emojis, the commands !adddefault [emoji] and !remdefault [emoji] will help as they will add/remove the emojis to Melon's DB and in the case that a message in the poll channel does not have any emojis, those will be added.

If no emojis were found in the message, generic ones will be added.

If there are emojis in the message, Melon will add those.

General Utilityüîó

Alongside the previously elaborated commands Melon has some miscellaneous utility commands which have been proven useful in the past:

!avatar [user]: This command will return the given user's (if not specified, your own) avatar in either a png or URL format (for animated avatars).

!rand [options]: If the option parameter follows the format [int]-[int] Melon will generate a random number, otherwise she will split the options with a space between them and return a random pick.

Colorüîó

!color [amt, tolerance] is a very useful command for when you want to setup roles for example or even when you're doing visual work such as web-design and need a color-palette. By attaching an image to the command Melon will know to turn that image into a color palette, with the default amt set to 4, if you need more you can always increase that and the default tolerance set to 6.

If you do not attach an image, Melon will scan the last 25 messages for an attachment, though currently URLs are not supported, you can use !color in combination with !avatar to get the color palette of someone's avatar.

Color will show you the most used colors in an image.
