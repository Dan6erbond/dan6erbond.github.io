Version: 1.5

Banhammer.py is a framework written for the Discord.py that allows you to create your very own Banhammer!

By injecting workloads into the asyncio event loop of Discord.py's bot, similarly to how a discord.ext.Cog would, it will stream new and reported items do your Discord bot. In addition it will handle adding reactions, performing their configured actions and of course, creating messages!

With its modular design you can take over as much as you want or stay minimalist and let Banhammer.py do the work. Banhammer.py is open-source, which means we'll be accepting pull requests on its GitHub repository!

class banhammer.reaction.ReactionHandlerğŸ”—

The default object that handles reactions unless a different implementation is given to Banhammer. 

handle (reaction, item, payload)ğŸ”—

Calls ReactionHandler.gen_handle() in its default implementation. This function should not be called directly, but instead over Reaction.handle().

Parameters

  • reaction (Reaction) - The reaction that contains the information of what actions should be performed.
  • item (RedditItem) - The item that is being handled.
  • payload (ReactionPayload) - The payload that will be fed with information.

Returns

ReactionPayload - The payload that was returned by Reaction.handle(). 

gen_handle (reaction, item, payload)ğŸ”—

The default implementation of reaction-handling that can remove, approve, lock, reply to, set the flair and mark posts as NSFW as well as ban the author permanently/temporarily.

Uses the information in reaction as well as the type of item to determine which actions to perform. Parameters

  • reaction (Reaction) - The reaction that contains the information of what actions should be performed.
  • item (RedditItem) - The item that is being handled.
  • payload (ReactionPayload) - The payload that will be fed with information.

Returns

ReactionPayload - The payload that was given and fed with information.

ReactionPayloadğŸ”—

class banhammer.ReactionPayloadğŸ”—

A data class that holds the actions performed on an item as well as other information and can be overriden to return custom messages/embeds.

Attributes

Attribute Description
item The item that is being handled by the reaction.
user The user that is performing the actions. Defaults to "Banhammer".
actions A list of type(str) that contains all the actions that were performed on the item.
approved A boolean, whether the item was approved or removed. Only applicable for submissions and comments.
reply The reply that was sent to the item, if applicable.
emoji The emoji that was used to perform the action.
