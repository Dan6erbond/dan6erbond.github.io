Banhammer.pyğŸ”—

Version: 1.7

Banhammer.py is a framework written for the Discord.py that allows you to create your very own Banhammer!

By injecting workloads into the asyncio event loop of Discord.py's bot, similarly to how a discord.ext.Cog would, it will stream new and reported items do your Discord bot. In addition it will handle adding reactions, performing their configured actions and of course, creating messages!

With its modular design you can take over as much as you want or stay minimalist and let Banhammer.py do the work. Banhammer.py is open-source, which means we'll be accepting pull requests on its GitHub repository!

class banhammer.Banhammer (reddit, loop_time=5 * 60, bot=None, embed_color=banhammer_purple, change_presence=False,
                           message_builder=MessageBuilder(), reaction_handler=ReactionHandler())ğŸ”—

Represents a banhammer instance that will work with the Discord.py and PRAW to interact with Reddit and Discord. A number of options can be passed to Banhammer, some of which are optional.

Parameters

  • reddit (praw.Reddit) - The Reddit instance with which actions will be performed.
  • loop_time (Optional[int]) - The number of seconds Banhammer should wait between loops, should be at least 120 to avoid rate-limits. Defaults to 300 to give the bot time to handle user input as PRAW blocks asyncio event loops.
  • bot (Optional[discord.Client]) - The bot that Banhammer is working for, only required if you need Banhammer to change the bot's presence.
  • change_presence (Optional[boolean]) - Whether Banhammer should change the bot's presence to "Watching Reddit" when it's scanning for items. Defaults to False.
  • message_builder (Optional[MessageBuilder]) - The MessageBuilder that Banhammer should use to generate item messages.
  • reaction_handler (Optional[ReactionHandler]) - The ReactionHandler that should be used when Reaction.handle() is called.

Event ReferenceğŸ”—

There are two ways of registering events to your bot. In either case the function that the event is registered will be called whenever there's a new item. Whenever the event is triggered, the function that it's assigned to call will receive whatever the event "returns" so make sure that is added to the function's parameters.

The first way is by using the decorators once an instance of Banhammer has been created, such as Banhammer.new(). The second way is by manually feeding all the arguments with the functions provided above such as Banhammer.add_new_func(). Following events are currently available in the Banhammer class:

  • Banhammer.new()
  • Banhammer.comments()
  • Banhammer.reports()
  • Banhammer.queue()
  • Banhammer.mod_actions()

ReactionğŸ”—

class banhammer.Reaction (**kwargs)ğŸ”—

Represents a reaction that is loaded from the wiki or local files by feeding it **kwargs. Dictionaries can be converted to **kwargs by using **dict_name.

Parameters

  • emoji (str) - The emoji that this reaction represents.
  • type (str) - One of either "submission", "comment", "modmail" or left empty. Thi is used when determining if the reaction is applicable for a certain item.
  • flair (str, Submissions only) - The flair that should be applied to a post.
  • approve (boolean) - Whether the post should be approved or removed. Defaults to False.
  • mark_nsfw (boolean, Submissions only) - Whether the post should be marked NSFW, defaults to False.
  • lock (boolean) - Whether the post should be locked by the bot. Is set to True of approve is set to False.
  • reply (str) - The text to reply to the post with. Can be left empty.
  • distinguish_reply (boolean) - Whether the reply should be distinguished (if one is present).
  • sticky_reply (boolean) - Whether the reply should be stickied (if one is present).
  • ban (int) - Whether the post author should be banned, None represents no ban while 0 represents a permanent one and every other number is a temporary ban of X days.
  • archive (boolean, ModMail only) - Whether the modmail conversation should be archived.
  • mute (boolean, ModMail only) - Whether the modmail's participant should be muted for 72 hours.
  • min_votes (int) - The minimum amount of votes required for the reaction's configured actions to be performed.
Attributes

Attribute Description
config All the arguments given to **kwargs stored in the dictionary format.
emoji The emoji assigned to this reaction, should be a unicode emoji or follow the Discord custom emoji format for bots.
type The type of items this reaction is designed to handle. Used in Reaction.is_eligible().
flair Whether the submission should be approved.
approve Whether the post should be approved. Defaults to False.
lock Whether the post should be locked.
reply Whether the post should be replied to.
distinguish_reply Whether the reply (if given) should be distinguished.
sticky_reply Whether the reply (if given) should be stickied.
ban If the post's author should be banned. None if no ban or 0 for a permanent ban.
archive Whether the modmail conversation should be archived.
mute Whether the modmail's participant should be muted for 72 hours.
min_votes The minimum amount of votes required for the reactino's configured actions to be performed.
item The RedditItem assigned to this reaction if it was retrieved through Reaction.get_reaction() or Reaction.get_reactions() otherwise None.

The default implementation of reaction-handling that can remove, approve, lock, reply to, set the flair and mark posts as NSFW as well as ban the author permanently/temporarily.

  • item (RedditItem) - The item that is being handled.
  • payload (ReactionPayload) - The payload that will be fed with information.
