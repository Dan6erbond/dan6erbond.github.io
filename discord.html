Generalüîó

Creating a serverüîó

Creating a server is easy! Use this big button to get started and then follow Discord's official tutorial to create a basic server. Make sure you select a server region that's close to where you are to reduce lag and ping times:

Starting creating a server with this button which can be found below your server-list.

After that you'll be greeted by a server with two channels: #general and #voice

Now that you have your very own server, it's time to customize it. Follow the steps below to get the server exactly where you want it!

Accessing the server settingsüîó

Now that you have your own server, you'll probably want to take a look at its settings to edit stuff like the icon and roles. For that click on the icon to the right of your server's name in the sidebar (left of the chat box) and select 'Server Settings':

The dropdown allows you to acess the server's setting and more.

Setting up channels and categoriesüîó

Channels and categories are the most important part about a server. They'll help you keep your server organized and separate topics and most importantly, allow you to hide topics based on permissions.

To create a channel or category, right-click on the sidebar of your server and select the one you wish to create:

Right-click on your server's sidebar to create a channel or category.

Creating a channel will ask you if you want to create a text- or voice-channel. As you'd expect, text-channels don't allow users to connect with each other over Discord's voice-chat and vice-versa.

When creating either of the two, you additionally get the option to make it 'private'. Selecting that option will show you a list of roles for which the channel/category will then be visible. Everyone else won't have read/send access to the newly created item:

If you're unhappy with the position of the newly created channel or category, you can hold and drag it around with your mouse and drop it to the desired position.

Categoriesüîó

Categories are like containers for your text- and voice-channels. They allow you to group relevant ones together and setup permissions that will be taken over by the category's channels unless you wish to override them.

Dragging a channel below a category will automatically make it a child of the category and if the permissions of the channel don't align with those of the category, Discord will offer you to 'sync' them which means to take over those permissions of the category. If the permissions aren't synced, the original permissions of the channel will be kept.

Channelsüîó

Permissions, invites and webhooks are topics that will be discussed in further detail in their own sections. This section focuses on the specific options a text-/voice-channel has that can be configured.

Text Channelsüîó

Text channels are where users can post, well, text! Hovering over a text-channel will reveal a tooth-weel that allows you to configure it:

Hovering over a channel allows you to enter its settings.

There are a few key settings you'll want to look at and configure to your liking:

  • Topic: This is basically a channel's description. It supports regular characters as well as unicode emojis.
  • Slowmode: Slowmode, when enabled, will limit a user's message-rate. If it's set to 5 seconds, users can only send one message every 5 seconds. This can be quite useful for servers with lots of members.
  • NSFW: If a channel is dedicated to 18+ content, it's recommended to toggle this option as it will warn users before they join the channel.

Rolesüîó

Roles are essential to an organized server. They allow you to distinguish members, assign colors and of course, configure permissions for each member. Roles can be created and configured in the server's settings under 'Roles'.

In the role configuration menu you'll see a list of roles to the left and the selected role's settings to the right. @everyone is the default "role" everyone has, it cannot be configured in terms of colors/name, but its permissions can be altered which we will go into more detail later.

Selecting any other role will allow you to edit its name, which supports emojis just like channels/categories do and its color. A handy method of getting HEX color codes is by using Melon's !color command which will show you the most common colors in an image.

Additionally you can decide whether a role should be mentionable and distinguished. Generally you only want people to be able to mention roles that serve a functional purpose. A good example for that are the moderators or developers.

Distinguishing a role means in addition to the name of a user with the role receiving the role's color, the user will be moved into a separate category on the user-overview.

If a user has multiple roles, they will be take over the color as well as be distinguished based on the role that's the highest in the list.

In terms of permissions, Discord will assign a user's permissions based on his/her highest permissions. Meaning that if a user has three roles, one with the 'Manage Messages' permission, the user will have those permissions where they aren't configured otherwise.

Permissionsüîó

Discord has a fairly complex system that allows server admins to limit the members' capabilities within a certain space. There are three levels in the permission system, each of them can be overriden by a lower level:

  • Guild
  • Category
  • Channel
This means that it's recommended to configure a role/member's permissions on a guild-level and just override from there.

Overridesüîó

Now that you know what permissions can be assigned to a user/role, it's time to setup overrides.

Overrides are simple once you've gotten the hang of them, the basic idea is that you have guild-wide permissions that will be used most of the time and those are overriden in the cases where different settings are required.

This usually means you'll be more restrictive on the guild-level and give users limited permissions for a channel or category.

As previously mentioned, once a user has a permission, it doesn't matter what other roles the user has. This means that basic permissions should be setup with @everyone, namely those that, well, everyone should have.

On the guild-level permissions can only be enabled/disabled. To override them, you go to a category or channel's settings and change the permission for a user/roles. By default the permission is set to "/", if you want to enable it for a certain area, it will need to be set to ‚úî and to disable it, change it to ‚ùå.

For that you right-click on the channel, select 'Edit Channel' and then go to the 'Permissions' tab. Over there you can add a new role to the list, in this case the @coders role and assign them the 'Manage Messages' permission by setting it to ‚úî.
  2. You don't want most of the server to be visible to people without the @verified redditors role. On the guild-level you'll want to disable the 'Read Messages' permission for @everyone and then enable it for @verified redditors.
